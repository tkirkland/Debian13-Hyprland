# Hyprland Welcome App — Design

**Date:** 2026-06-12
**Issue:** [#11](https://github.com/tkirkland/Debian13-Hyprland/issues/11) — Add missing Hypr dependency "Hypr Welcome"
**Status:** Approved

## Problem

The welcome app (`hyprland-welcome`) is already built and installed: it lives
at `utils/welcome` in hyprland-guiutils (present in the pinned v0.2.x release
tags; the root CMakeLists builds every util and `cmake --install` puts the
binary at `/usr/local/bin/hyprland-welcome`). What's missing is the launch:
the installer generates its own `hyprland.lua`, replacing Hyprland's stock
autogenerated-config flow that would normally surface the welcome window. The
user never sees it.

## Decisions

- **First login only**: the welcome window appears on the user's first
  Hyprland session and never again.
- **Shell-guarded exec-once** in the generated config (not Lua-side file
  logic): a marker file `~/.config/hypr/.welcome-shown` gates the launch and
  is touched even if the app exits nonzero, so a broken welcome app can
  never nag every login.
- **Absolute path** `/usr/local/bin/hyprland-welcome`, matching repo
  convention.

## Components

### 1. Generated config (scripts/60-hyprland.sh, hyprland.lua heredoc)

Add to the generated `hyprland.lua` an exec-once entry equivalent to:

```
sh -c '[ -f ~/.config/hypr/.welcome-shown ] || { touch ~/.config/hypr/.welcome-shown; /usr/local/bin/hyprland-welcome; }'
```

with a short comment explaining the first-login marker. Marker is touched
BEFORE the app runs (crash-proof nag guard).

### 2. Verify phase (scripts/90-verify.sh)

New vcheck: `/usr/local/bin/hyprland-welcome` is executable in the target —
in the in-chroot-build path only (the `else` branch of the
`BUILD_ON_FIRSTBOOT` conditional, alongside the existing "Hyprland binary
runs" checks; in firstboot mode the binary does not exist at verify time).
This is the guard for the issue's "current build lacks this": if a future
guiutils tag drops or renames the util, verification fails loudly.

## Testing (tests/hyprland-config.sh)

- Generated lua contains the welcome exec line, the marker path, and the
  absolute binary path.
- Verify body contains the new vcheck label.

## Out of scope

- Building anything new (guiutils already builds welcome).
- Hyprland's own autogenerated-config welcome heuristics.
- The donate/update screens shipped alongside welcome in guiutils.
